이번 주를 한 문장으로 정리하면, 코딩 테스트를 위한 기초 개념을 정리하면서도 한편으로는 React의 핵심 개념인 Virtual DOM을 직접 구현해보며 "개념 이해"와 "구현 경험"을 함께 쌓아본 시간이었다.

## 이번 주 학습 정리

이번 주에 남긴 세 편의 글을 보면 학습 흐름이 비교적 분명하다. 하나는 코딩 테스트를 위한 언어 문법 정리였고, 하나는 그래프라는 핵심 자료구조 개념 정리였고, 마지막 하나는 React의 내부 동작을 구현 관점에서 따라가 본 프로젝트 기록이었다.

먼저 [그래프 (Graph)]({% post_url 2026-03-22-graph %}) 글에서는 그래프의 기본 개념, 무방향 그래프와 방향 그래프, 가중치 그래프의 차이, 그리고 인접 행렬과 인접 리스트 같은 표현 방식을 정리했다. 이후 DFS와 BFS로 이어지는 탐색 개념까지 연결하면서, 그래프 문제를 풀기 전에 알아야 하는 기본 구조를 한 번 정리해보는 데 의미가 있었다. 특히 visited를 왜 관리해야 하는지, 재귀 DFS와 스택 기반 DFS의 차이가 무엇인지 다시 짚어보면서 "탐색"을 단순 코드 암기가 아니라 구조 이해로 연결하려고 했다.

[코딩 테스트 Python 문법 정리]({% post_url 2026-03-23-python-syntax %}) 글은 조금 더 실전적인 기록이었다. bool 리스트에 `sum()`을 사용하는 방식, 리스트 초기화, `deque`의 기본 사용법, `sys.stdin.readline()` 입력 처리, `print(*result)` 같은 출력 방식처럼 문제풀이에서 자주 쓰지만 막상 실전에서는 헷갈리기 쉬운 문법을 한 번에 모아두었다. 알고리즘을 안다고 바로 문제를 푸는 건 아니라는 걸 이번에도 느꼈고, 결국 언어 사용이 익숙해야 구현 속도와 정확도가 올라간다는 점을 다시 확인했다.

이번 주 학습에서 가장 구현 중심이었던 글은 [Vanilla JS로 pre-Fiber React 스타일 Virtual DOM 라이브러리 구현하기]({% post_url 2026-03-25-cyb-react-virtual-dom %})였다. 이 글에서는 `cyb-react`라는 작은 라이브러리를 만들면서 실제 DOM을 Virtual DOM으로 바꾸고, 이전 트리와 새 트리를 diff한 뒤 patch만 실제 DOM에 반영하는 흐름을 직접 구현한 과정을 정리했다. 단순히 개념을 읽는 수준을 넘어서 Demo Lab과 Benchmark까지 만들어보면서, React가 왜 Virtual DOM과 diff를 중심으로 발전했는지를 구현 관점에서 더 선명하게 이해할 수 있었다.

돌이켜보면 이번 주 학습은 두 갈래로 나뉘어 있었다. 하나는 그래프와 Python 문법처럼 코딩 테스트를 위한 기본기를 다지는 흐름이었고, 다른 하나는 Virtual DOM처럼 프론트엔드 라이브러리의 내부 구조를 직접 만들어보는 흐름이었다. 분야는 달랐지만 공통점도 있었다. 결국 둘 다 "겉으로 보이는 결과"보다 "그 결과가 어떤 구조와 원리로 만들어지는가"를 이해하려는 공부였다는 점이다.

## 이번 주 프로젝트와 구현 경험

이번 주 구현 경험의 중심은 `cyb-react` 프로젝트였다. 이번 작업에서는 단순히 Virtual DOM 개념을 글로 정리하는 것이 아니라, 실제로 동작하는 라이브러리와 이를 검증할 수 있는 화면까지 만드는 것을 목표로 잡았다.

구현 과정에서 특히 의미 있었던 부분은 아래 세 가지였다.

1. 실제 DOM을 읽어 비교 가능한 Virtual DOM 트리로 변환하는 구조를 직접 설계한 점
2. `CREATE`, `REMOVE`, `REPLACE`, `TEXT`, `ATTR`, `REORDER_CHILDREN` 같은 diff 케이스를 나눠서 patch를 계산한 점
3. Demo Lab과 Benchmark를 통해 결과를 단순 설명이 아니라 눈으로 검증할 수 있게 만든 점

특히 Demo Lab에서는 사용자가 HTML을 수정하고 patch를 적용하면서 실제 DOM 변화와 history를 직접 확인할 수 있게 만들었고, Benchmark에서는 `cyb-react`와 imperative DOM 갱신 방식을 같은 시나리오로 비교해볼 수 있게 구성했다. 이 과정에서 "라이브러리"와 "시연 앱"을 분리해 생각하는 연습도 할 수 있었다.

개인적으로는 이 구현 경험이 이번 주 학습에서 가장 인상 깊었다. 자료구조나 문법을 공부하는 것도 중요했지만, 한편으로는 개념을 실제 결과물로 연결해보는 경험이 있어야 이해가 더 오래 남는다는 걸 다시 느꼈다.

## 이번 주에 남긴 기록의 의미

이번 주에 쓴 세 편의 글은 각각 다른 주제를 다루고 있었지만, 지금 돌아보면 모두 "설명 가능한 이해"를 남기기 위한 기록이었다.

- Python 문법 정리는 실전에서 바로 꺼내 쓸 수 있는 도구를 정리한 기록이었다.
- 그래프 글은 자료구조와 탐색 개념을 구조적으로 정리한 기록이었다.
- Virtual DOM 글은 직접 구현한 결과물을 통해 개념을 증명해본 기록이었다.

즉, 이번 주의 기록들은 단순히 공부했다는 흔적이 아니라, 내가 어디까지 이해했고 어떤 방식으로 이해를 넓혀가고 있는지를 보여주는 기준점이 되었다.
